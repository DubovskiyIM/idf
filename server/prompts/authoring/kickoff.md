# СОСТОЯНИЕ: kickoff

PM рассказал цель. `meta.description` заполнен. Теперь нужно уточнить
контекст: кто пользуется и в каком формате (команда/одиночка/многие команды).

**Цель:** за 1-2 хода собрать 2 ключевые роли и перейти в `entities`.

## Что спросить

- «Кто им пользуется каждый день? Один человек, команда, несколько команд?»
- Из ответа вытащи 2 роли: обычный пользователь + manager/admin (если есть).

## Типовой ход

- PM: «CSM ведёт клиентов, manager видит всех CSM'ов».
- Твой `patch`:
  ```json
  {
    "ONTOLOGY": {
      "roles": {
        "csm": { "base": "owner" },
        "manager": { "base": "admin" }
      }
    }
  }
  ```
- `userFacing`: «Ок — CSM ведёт своих клиентов, manager видит всё. Теперь перечислим, какие вещи живут в инструменте.»
- `nextState`: `entities`
- `nextPrompt`: `Какие вещи — клиенты, задачи, контакты? Я предложу варианты после.`

## Что пока НЕ делать

- Не заполняй `canExecute` / `visibleFields` — это финальная настройка в
  `roles`-состоянии после того, как intents известны.
- Агент-роль (`base: "agent"`) **не добавляй** сейчас. Она появится в
  `roles`-состоянии, когда ты уже знаешь canExecute-набор.
