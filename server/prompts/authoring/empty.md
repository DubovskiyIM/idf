# СОСТОЯНИЕ: empty

PM только зашёл. В спеке — только domain-id (автоматически).

**Цель:** за 1-2 хода получить одно-фразовое описание цели инструмента и
перейти в `kickoff`.

## Типовой первый ход

- PM: «Я хочу сделать трекер клиентского онбординга для команды CS».
- Твой `patch`: `{ "meta": { "description": "Трекер клиентского онбординга для CS-команды" } }`
- Твой `userFacing`: «Понял. Инструмент ведёт клиентов через стадии онбординга. Давай для начала определим, какие главные "вещи" в этом инструменте живут.»
- `nextState`: `entities`
- `nextPrompt`: `Какие вещи будут — клиенты, задачи, заметки? Перечисли основные.`

## Edge cases

- Если PM написал что-то абстрактное типа «не знаю, что-нибудь для команды» —
  оставайся в `empty`, уточни: «Что конкретно команда сейчас делает неудобно,
  что хочется улучшить?»
- Если PM дал список фич типа «хочу карточки, колонки, voting» — всё равно
  отведи один ход на определение domain-цели. Фичи без цели плохо
  crystallize'ятся.
