# Gravitino gap-каталог (Stage 1 baseline)

**Дата:** 2026-04-22
**Baseline commit:** `bb1facb` (после регистрации домена в `main.jsx`+`standalone.jsx`) + vite-config fix.
**Методология:** серверный `crystallizeV2` прогон по imported ontology (218 entities / 120 intents); stderr capture анкоринг-ошибок; HTTP smoke `https://localhost:5173/gravitino` → 200 (SPA-fallback работает после patch'а `vite.config.js`).
**Design reference:** `docs/superpowers/specs/2026-04-22-gravitino-dogfood-design.md` (8 стадий, horizontal-slice).

## Ключевое наблюдение Stage 1

**Из 120 intents crystallize_v2 деривирует только 3 artifact'а** (`lineage_create`, `metalake_create`, `run_create` — все form-архетипы). Нет ни одного catalog/detail/dashboard. Причина — сочетание факторов:

1. **Enrich не сработал** (§1.12 backlog'а) — pre-enrich ontology не обогащена role-inference / particles.
2. **200+ envelope-сущностей** (`*CreateRequest`, `*ListResponse` и т.д.) без ownerField / semantic-role — crystallize не может сопоставить intents с entity для catalog/detail.
3. **Anchoring §15 в soft-mode** — 187 suppressed errors; intents без particle-constraint не проходят anchoring-фильтр для heavy-архетипов.
4. **Intent α-distribution:** 3 insert + 47 replace + 17 remove + 53 read-only. 47 replaces = updates (PATCH), но crystallize не создаёт form для replace без матчащей detail-projection.

**Без Stage 2-3 работ (deep-tree nav + composite-types) baseline визуально почти пустой.** Это ожидаемое состояние — Stage 1 цель = «выявить gap'ы для следующих стадий», не «работающее демо».

## Структура gap-каталога

Каждый gap: `{id, severity, module, observation, target-stage, how-to-reproduce}`.
- **Severity:** P0 (блокирует demo) / P1 (визуально плохо) / P2 (cosmetic).
- **Target-stage:** Stage 2-7 (capability) / Deferred / SDK-backlog (вне Gravitino-спринта).

## Gap'ы

### ✅ G1 — Crystallize деривация + tree-nav (ЗАКРЫТО 2026-04-23)

**Severity:** P0 → закрыт.
**Final state:** 27 artifact'ов (12 catalog + 12 detail + 3 form); `hierarchy-tree-nav` pattern apply'ится на `metalake_detail` с полной Gravitino-иерархией в `sidebar[0]`.

**Тройной путь закрытия:**

1. **Stage 2 Task 0 — авторирование 24 projections** (12 list + 12 detail) по шаблону compliance. Разблокировало catalog/detail архетипы.
2. **PR idf-sdk#186 — enricher-claude@0.2.1** `structured_output` wire-format. Enrich теперь работает на Gravitino: +17 intents, +63 field-roles, +14 absorbedBy, 3 base roles (owner/admin/viewer).
3. **PR idf-sdk#188 — importer-openapi@0.5.0** path-derived FK synthesis. 28 entities получили `<parent>Id: { kind: "foreignKey", references: ... }` из URL-структур. Это разблокировало `hierarchy-tree-nav.findChildEntities` и R8 hub-absorption на path-based REST API.

**Tree-nav результат:**
```
Metalake → Tag, Catalog, User, Group, Role, Owner, Policy
Catalog  → Schema
Schema   → Table, Fileset, Topic, Model
Tag      → Object
...
```

**Related memory:** `docs/gravitino-ux-patterns-notes.md` P-1 (path-derived hierarchy) — кандидат в pattern bank researcher.

### G2 — 200+ envelope-типов в ontology.entities

**Severity:** P1
**Module:** importer pipeline
**Observation:** 218 entities из которых ~200 — DTO wrappers (`CatalogCreateRequest`, `ListResponse`, `NameIdentifier`, `Audit`, `ErrorModel`, 15+ Job/JobTemplate вариаций, etc). Canonical entities (12) — островки в этом шуме.
**Target-stage:** Deferred → SDK backlog (importer-openapi heuristic `isEnvelope` / `entity.kind: "envelope"`). См. Task 9 gap #20 в session backlog.
**Reproduce:**
```bash
node -e "import('./src/domains/gravitino/imported.js').then(m => {
  console.log(Object.keys(m.ontology.entities).filter(e => /Request|Response|Audit|Error/i.test(e)).length);
})"
```
**Mitigation для Stage 2+:**
- Patch host-facade `ontology.js` — фильтровать envelope'ы по regex `/Request|Response|...$/i`, экспортировать clean ONTOLOGY.
- Правильное решение — upstream importer heuristic.

### G3 — importer-openapi header говорит "postgres"

**Severity:** P2
**Module:** importer pipeline
**Observation:** `imported.js:1` — `// Generated by @intent-driven/importer-postgres at ...`, хотя реально отработал importer-openapi. Template-строка не проецируется по типу importer'а.
**Target-stage:** SDK backlog. Also see Task 9 gap #20 tracked.
**Reproduce:** `head -1 src/domains/gravitino/imported.js`.
**Mitigation:** 1-строчный fix в `@intent-driven/importer-openapi`: собственный serialize с правильным header'ом.

### G4 — Host registration fragmented (main.jsx + standalone.jsx + vite.config.js)

**Severity:** P1
**Module:** host
**Observation:** Регистрация нового домена требует правок **3-х файлов** минимум:
1. `src/main.jsx` — `<Route>`
2. `src/standalone.jsx` — 4 записи (import + DOMAIN_ADAPTERS + DOMAINS_RAW + DOMAIN_TITLES + DOMAINS)
3. `vite.config.js::spaFallback()::SPA_ROUTES` — hardcoded whitelist (иначе `/gravitino` → 404)

Пропустил `spaFallback` во время Task 7 implementation → потратили шаг на диагностику.
**Target-stage:** Stage 8 / SDK backlog.
**Reproduce:** GIT-archeology показывает что delivery/freelance/compliance тоже частично не зарегистрированы (см. host-debt observation).
**Mitigation:**
- Вариант A: SPA-fallback рефакторить на dynamic — читать DOMAINS из manifest, а не hardcoded list.
- Вариант B: регистрирующий helper `registerDomain({id, domain, adapter, route})` — single entry point.
- Вариант C: Studio-driven domain registration — но это M1.2+ scope, не Stage spring.

### G5 — Deep hierarchy nav отсутствует (Metalake→Catalog→Schema→Table)

**Severity:** P0 (cornerstone спринта — Stage 2)
**Module:** navigation shell
**Observation:** Даже если crystallize дериввирует catalog/detail для 12 canonical entities, current `TreeNav` primitive (renderer@0.28.0) не поддерживает lazy-expand / search / keyboard-nav / nested 4+ уровня. Gravitino требует классическую tree-navigation; IDF сейчас показывает flat-list projections.
**Target-stage:** Stage 2 (deep tree + breadcrumbs).
**Reproduce:** `/lifequest` или `/compliance` в браузере — видно плоский sidebar. `/gravitino` после G1 fix тоже будет плоский без Stage 2.
**Mitigation:** Stage 2 плана — расширение `packages/renderer/src/primitives/tree-nav/` + новый `Breadcrumbs` primitive.

### G6 — Composite/nested типы не рендерятся (varchar(N), decimal(P,S), list<T>, struct<...>)

**Severity:** P0 (cornerstone — Stage 3)
**Module:** form archetype / parameter kinds
**Observation:** `ontology.entities.Table.fields.columns` — array of nested structures с типами `varchar(100)`, `decimal(10,2)`, `list<int>`, `struct<...>`. Сейчас renderer'а `composite-type` parameter нет, `SchemaEditor` primitive отсутствует.
**Target-stage:** Stage 3 (composite + nested types).
**Reproduce:** Когда G1 исправлен и `Table_create` дериввируется — form будет пытаться ренедерить Array<Object> как JSON-textarea (или вовсе пропустит).
**Mitigation:** Stage 3 — `composite-type` parameter kind + `SchemaEditor` primitive.

### G7 — Data-grid (MUI X-DataGrid уровень) отсутствует

**Severity:** P1 (Stage 4)
**Module:** catalog archetype rendering
**Observation:** Gravitino WebUI использует @mui/x-data-grid — column resize, pinning, filter-popover per column, grouping, virtualized 1000+ rows. Наш catalog-архетип (через AntD Table) поддерживает только sort/pagination.
**Target-stage:** Stage 4.
**Reproduce:** Любой catalog-projection в invest/compliance — сравнить с Gravitino catalog screenshots.
**Mitigation:** Stage 4 — новый `DataGrid` primitive поверх AntD Table + rc-virtual-list.

### G8 — Permission matrix для Access-модуля

**Severity:** P1 (Stage 5)
**Module:** Access module / role-capability rendering
**Observation:** Gravitino Access имеет grid role × capability × entity-type с granular grant/revoke. IDF `ontology.invariants[].role-capability` даёт declarative данные, но renderer-primitive для матрицы нет.
**Target-stage:** Stage 5.
**Reproduce:** Compliance имеет 6 ролей и 15 invariants с role-capability — но нет матрицы в UI, только скрытые toggle'ы в detail.
**Mitigation:** Stage 5 — новый `PermissionMatrix` primitive.

### G9 — Multi-step wizard с provider-specific config

**Severity:** P1 (Stage 6)
**Module:** wizard-archetype
**Observation:** Gravitino CatalogCreate — multi-step wizard, шаги зависят от `provider` (hive/iceberg/jdbc/...). Наш `discriminator-wizard` pattern даёт dynamic shape'ы, но не dynamic step-graph.
**Target-stage:** Stage 6.
**Reproduce:** G1 fix → `metalake_create` derived, но без multi-step structure.
**Mitigation:** Stage 6 — расширение `discriminator-wizard.apply` + новый control kind `test-connection` (async validation).

### G10 — Property popover + tag/policy associations

**Severity:** P2 (Stage 7)
**Module:** detail + associations
**Observation:** Gravitino использует property popover (hover-trigger) на metadata-rich сущностях; tag/policy attach/detach через chip-UI. В IDF — `keyboard-property-popover` matching-only (не apply), `TagChip`/`PolicyChip` primitives отсутствуют.
**Target-stage:** Stage 7.
**Reproduce:** После G1 — `Table_detail` с tag-ассоциацией не покажет chips inline.
**Mitigation:** Stage 7 — `PropertyPopover` primitive + `TagChip`/`PolicyChip` + `tag-policy-manager` pattern.

### G11 — 187 anchoring errors suppressed (soft-mode)

**Severity:** P1
**Module:** crystallize anchoring §15
**Observation:** Stderr во время crystallize: `[crystallize_v2] unknown: 187 anchoring errors suppressed (soft mode)`. Это intents без particle-constraints — anchoring-фильтр помечает их unreliable, но soft-mode не блокирует.
**Target-stage:** SDK backlog + частично Stage 1-bis (после enrich fix).
**Reproduce:** `console.error` capture при `crystallizeV2` запуске.
**Mitigation:**
- enricher-claude должен proactively добавлять particles на importer-output (§1.12 backlog).
- OR importer-openapi должен генерировать placeholder-particles из operationId semantics.

### G12 — `importer-openapi` duplicate schema rename (`Job` → `Job-2`)

**Severity:** P2
**Module:** importer/redocly
**Observation:** Redocly rename'ил конфликтующий schema `Job` → `Job-2` во время bundle. Importer сохранил `Job-2` как orphan entity без intents. Не блокирует (intents на canonical `Job` работают), но засоряет entity-list.
**Target-stage:** SDK backlog.
**Reproduce:** `grep "Job-2" src/domains/gravitino/imported.js`.
**Mitigation:** importer-openapi эвристика «если entity без intents после всех путей — drop».
