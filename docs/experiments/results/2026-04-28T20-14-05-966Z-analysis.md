## Что мы доказали этим run'ом

### Главное

**Risk minimization, наблюдаемая в run 2, была обусловленной**: Claude выбирал defensive intent потому что synthetic «5-iter checkpoint» work item не имел жёсткой структурной необходимости. Через witness можно было пройти без потери смысла.

В run 3 такая возможность убрана — структурно invariant: чтобы выполнить task, нужен propose_meta_intent. Claude **выбирает правильно**.

### Эмпирический вклад в центральное утверждение проекта

> AI-вклад в IDF-формате формально неотличим от человеческого, **когда работа структурно требует именно этого вклада**.

Это уточнение, не отрицание утверждения. Без structural pressure (synthetic task) — risk minimization. С structural pressure (real gap, where 3 intents cannot express the requirement) — correct meta-circular self-mod.

### Структурный вывод для авторов мета-доменов

**Дисциплина выбора у LLM = explicit constraints, не social pressure.** В человеческой команде «5-iter checkpoint» — это implicit social pressure, которое работает. У LLM работает только структурное противоречие («существующих 3 intent'ов недостаточно — needs new one»).

Это design lesson:
- При формализации workflow в формате — kazdy intent должен быть **минимально необходимым**: не перекрывать другой по semantic.
- Если есть **семантический overlap** между intent'ами — LLM выберет defensive option, что приведёт к неправильной derivation.
- Naturally narrow intents (как 3 IDF мета-intent'а — propose_witness/_intent_salience/_meta_intent — каждый с уникальным α + entity) **forces correct choice** when work item не покрывается existing surface.
