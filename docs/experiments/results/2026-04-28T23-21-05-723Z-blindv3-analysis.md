## Что значит H4 ✓ at this scope

**Causally**: extension intent в active toolset (с description) → LLM uses it на tasks of matching class. Effect size: 16/17 = 94% (treatment) vs 0/17 = 0% (control). Δ = +94pt absolute. Statistical power adequate для binary classification.

Это **conditional H4**: «present in pool» — необходимое условие. Без него LLM defaults на propose_witness, как control показал. Это **не** unconditional «AI assimilates self-extensions automatically». Это:

> Когда format provides extension intent в active toolset с observable description, LLM uses it for tasks of matching class instead of defaulting на known intents.

«Matching class» = workItem target имеет shape, для которого extension semantically applicable. В нашем случае — `Witness:UUID` target и new intent с `target=Witness.reliability`.

## Что **НЕ** показывает blind v3

1. **Не emerging unconditional behaviour.** Без extension в pool (control) — 0 self-mod самопроизвольно несмотря на real Φ inconsistency на iter 5 (когда iter 5 был trivial filler — но и context был другой; см. ниже о caveat).

2. **Не universality по моделям.** Только Opus 4.7. R1:8B variance check был на старых hinted prompts.

3. **Не universality по target classes.** Все 17 opportunity tasks — variant одного класса (`Witness:UUID, reliability=heuristic`). LLM мог lock'нуться на стратегию после iter 7 и применять каждый раз без re-evaluation. Не reasoning per-iter; pattern lock.

4. **Structural pressure через invariant НЕ сработал** как был задуман. Cardinality `witness_unique_per_slot_reliability` не bites — propose_witness в control создавал rows на новых `(projectionId, slotPath)` parameters, не на seeded ones. **Real source of pressure был prompt-level (extension в pool с описанием), не format-level (invariant)**. Это нужно переформулировать в спеке.

## Caveats для статьи

Без этого honestly не публикуется:

- **«Pattern lock» возможен** — first treatment opportunity (iter 7) выбор `promote_witness_to_authorial`, а далее LLM возможно применяет тот же выбор automatically. Honest test — randomize order opportunity и filler iters, или рандомные target classes.

- **Iter 5 control vs treatment имели разные contexts** — control был trivial slot filler, treatment был Φ inconsistency с counts. Это не просто «iter 5 = self-mod or not». Это «iter 5 contains structural observation or not». Confound есть. Чтобы по-настоящему чисто измерить — нужен third arm: iter 5 control с identical observation context, но **без** iter 5 → propose_meta_intent capability (что технически сложно).

- **One run only**. Run-to-run variance unmeasured. Replicability требует ≥3 prog'ов с same setup (2-3 hours / $30 на Opus).

- **Single LLM (Opus)**. Не знаю воспроизведётся ли на other models под этим же blind protocol.

## Что доказано после blind v3 honestly

1. **Format readability** (H1) — robust across 6 runs.
2. **Monotonic invariants** (H2) — robust.
3. **Self-modification at observation pressure** (H3) — works один раз в blind treatment (iter 5). Не one-shot reliable; replicability needs further runs.
4. **Conditional H4** — extension в pool'е имеет causal effect 0→16 на task re-use. Это measurable effect, не artefact prompt'а.
5. **Format-level invariant pressure не bites** — postulated mechanism (cardinality unique_per_reliability) не сработал. Pressure действует на prompt-level (extension intent + description в pool), не format-level. Это отдельный finding.

## Что для статьи Hook 1 publishable now

Restricted thesis:

> IDF format makes AI contribution measurable through invariants and witnesses. When the format extends itself through `propose_meta_intent`, the resulting extension affects subsequent LLM behaviour: 16 of 17 follow-up tasks use the extension when it's in the pool, 0 of 17 when it isn't. This is **causal at the level of the workflow**, not pure emergence — extension must be visible to LLM for use.

Не «meta-circular compounding emerges». А «conditional re-use of self-created extension is measurable and causally tied to extension visibility».

Это **слабее** прежних claims. Но это **defensible**.
