The Atom
Five particles. Complete decomposition.
Every user intention decomposes into exactly five particle types. This covers the full cycle of intentional action: what exists, when applicable, what changes, what to show, how to confirm.
C
Conditions
When is this action available? Predicates on world state. Static or temporal.
F
Effects
What becomes true? The only substance of state. Add, replace, remove, batch.
W
Witnesses
What does the actor need to know? The human-readable dual of conditions.
P
Confirmation
What does the system need back? Click, text, form, or custom.
→
Derivation
From particles, the system derives everything: projections, workflows, algebra, UI, API.

Intent Algebra
Relations derived, not declared.
Four relation types emerge automatically from intention particles. The graph is recomputed on every change.
▷

Sequential

Effect of I₁ enables condition of I₂. Workflow transitions emerge from this.

⇌

Antagonistic

Applying both returns to prior state. Publish / unpublish. Archive / restore.

⊕

Excluding

Effects conflict by composition table. Algebraically incompatible.

∥

Parallel

Shared entities, no conflict. Safe to execute independently.

Developer Experience
Define. Derive. Done.
  app.js
import { fold, computeAlgebra, deriveProjections } from '@idf/core';

// Define what users come to do
const INTENTS = {
  create_task:   { creates: "Task(draft)",   confirmation: "enter" },
  publish_task:  { conditions: ["task.status = 'draft'"],  effects: ["status → active"] },
  complete_task: { conditions: ["task.status = 'active'"], effects: ["status → done"] },
};

// World is computed, never stored
const world = fold(effects, typeMap);

// Relations emerge automatically
const algebra = computeAlgebra(INTENTS, ONTOLOGY);
// create ▷ publish ▷ complete  — derived, not declared

// Screens emerge automatically
const projections = deriveProjections(INTENTS, ONTOLOGY);
// → task_list (catalog), task_detail (detail)  — 78% coverage
