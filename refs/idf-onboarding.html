Слайд 1 · Позиционирование

IDF — формат, не фреймворк

Intent-Driven Frontend описывает приложение декларативно: онтология, намерения, проекции, роли. Это тип данных — уровня OpenAPI или JSON-LD. Адаптеры и материализаторы читают формат.

Один артефакт · четыре читателя

§2 · Φ

Source of truth — история намерений

Мир = fold(Φ_confirmed). Мир не хранится — он перевычисляется. Каждое намерение проходит жизнь: proposed → confirmed | rejected.

§17 · адаптеры

UI-кит — поздняя привязка

Проекции не знают про React-компоненты. Адаптер подставляет Button, Text, Card в runtime через capability-surface.

§15 · анкеринг

LLM — на этапе авторства

Модель помогает конструировать артефакт и кристаллизовать UI. В рантайме решения принимает детерминированное ядро.

Главный тезис: сначала описываем смысл приложения, потом материализуем — в пиксели, голос, агента, документ.
Слайд 2 · Как это работает в рантайме

Намерение → эффекты → мир

Пользователь (или агент) формирует intent. Валидатор проверяет онтологию, инварианты и гварды. На подтверждении рождаются эффекты — они попадают в Φ. Мир пересобирается свёрткой.

Конвейер одного намерения

Intent (декларация)

// src/domains/freelance/intents.js
confirm_deal: {
  params: ["executorId","taskId","responseId","amount"],
  roles: ["customer"],
  effects: [
    { alpha: "create", entity: "Deal",       /* +escrow */ },
    { alpha: "create", entity: "Transaction",kind:"escrow-hold" },
    { alpha: "replace",entity: "Wallet",      /* balance-=, reserved+= */ }
  ],
  context: { __irr: { point: "high" } }  // §23 irreversibility
}

После confirm — что появилось в Φ

// engine · fold · применяет α-операции к миру
Φ.push({ tau:"effect", alpha:"create", entity:"Deal", data:{
  id:"d_42", status:"in_progress", amount:9000, commission:900
}})
Φ.push({ tau:"effect", alpha:"replace", entity:"Wallet",
  patch: { balance: -9000, reserved: +9000 }})

// мир после fold:
world.Deal.d_42 = { status: "in_progress", ... }
world.Wallet.w_a = { balance: 11000, reserved: 9000 }
Ключевое: декларация → валидация → эффекты → Φ → мир. Ни один шаг не «додумывает» — всё следует из артефакта.

Слайд 4 · Как разрабатывать

Рецепт домена — пять шагов

От онтологии к живому приложению. Цикл занимает от часа (smoke) до пары дней (Cycle 1 с escrow). Референс: src/domains/freelance — биржа услуг, Cycle 1 + escrow core.

  1. Онтология. ontology.js: entities (internal / reference / mirror / assignment), роли с base, ownerField, role.scope для m2m, инварианты (invariants[] — 5 kinds).
  2. Намерения. intents.js: params, roles, effects (α: create | replace | remove), context (__irr для irreversibility §23). Guards — чистые функции.
  3. Проекции. projections.js: 4–8 проекций, 4 из них ROOT. Архетип можно не указывать — кристаллизатор выведет сам.
  4. Правила. rules.js: event-condition-action. Расширения v1.5 — aggregation / threshold / schedule / condition. Scheduler хранит таймеры в Φ.
  5. Seed + тесты. seed.js + *.test.js: ontology, intents, projections, filterworld, escrow. Ручной smoke в MANUAL_SMOKE.md.

Структура домена

// src/domains/freelance/

ontology.test.js              // ∀-свойства онтологии
intents.test.js               // структура + irreversibility markers
projections.test.js           // архетипы · ROOT · subCollections
filterworld-multi-role.test.js// viewer-scoping по ролям

MANUAL_SMOKE.md               // 7 happy + 4 negative шага
README.md                     // working-state · границы · open items

Слайд 5 · Студия и экосистема

Авторская среда, SDK и следующий шаг

IDF живёт в двух репозиториях: прототип (host + 9 доменов) и SDK-монорепо (@intent-driven/core · renderer · adapter-* · canvas-kit · cli). Студия даёт авторский инструментарий: Graph3D, Pattern Inspector, explain-match.

Экосистема инструментов

Студия (§27)

  • Graph3D — визуализация онтологии и связей m2m
  • Pattern Inspector — Cmd+Shift+P: Off / Preview / Commit для structural patterns
  • explainMatch — единая SDK-surface: per-requirement breakdown триггера
  • Claude proxy — dev-time :4000, роль — автор, не рантайм
  • Researcher — scripts/pattern-researcher.mjs, self-improving Pattern Bank

Банк паттернов — 13 stable, из них 3 с structure.apply. Остальные 10 в roadmap v1.13+.
